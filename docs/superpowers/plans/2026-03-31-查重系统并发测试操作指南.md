# 查重系统并发测试操作指南

## 1. 文档目标

本文档用于指导从未做过测试的开发者或项目成员，从零开始完成一次针对当前查重系统的并发测试。

本次测试目标不是做复杂压测平台，而是完成一条可落地、可复现、可展示的测试链路：

1. 启动系统
2. 准备测试数据
3. 模拟 100 个学生提交作业
4. 教师发起查重
5. 观察任务执行过程
6. 核对数据库结果
7. 记录测试结论

---

## 2. 本次测试要验证什么

本次测试主要验证两部分能力：

### 2.1 提交阶段并发能力

验证 100 个学生在短时间内集中提交作业时，系统是否能够稳定完成：

- 登录
- 创建 `submission`
- 保存 `submission_file`
- 解析 Java 文件
- 生成 `submission_profile`

### 2.2 查重阶段处理能力

验证教师在 100 份有效提交完成后发起一次查重任务时，系统是否能够稳定完成：

- 创建 `plagiarism_job`
- 执行 4950 组代码对比较
- 更新任务进度
- 保存 `similarity_pair`
- 保存 `similarity_evidence`
- 正常结束为 `DONE`

说明：

- 100 个学生同一班级下的两两比较数为：

```text
100 * 99 / 2 = 4950
```

---

## 3. 当前系统并发特点说明

根据当前后端实现，查重系统的并发特点如下：

### 3.1 学生提交

学生提交接口支持并发请求，主要压力点在于：

- 接口接收请求
- 文件保存
- Java 解析
- 数据库写入

### 3.2 教师发起查重

教师发起查重时，会创建一个异步 Job：

- 接口路径：`POST /api/teacher/assignments/{assignmentId}/plagiarism-jobs`

当前实现中：

- Job 本身是异步执行
- 单个 Job 内部的 4950 组比较是顺序计算
- 多个 Job 的并发能力受线程池限制

当前线程池默认配置：

- core pool size = 2
- max pool size = 4
- queue capacity = 20

所以当前测试最适合先做：

1. 100 个学生并发提交
2. 教师发起 1 个大 Job

如果后续要进一步测试任务并发，再补“多个作业同时查重”。

## 10. 第五步：学生并发提交脚本

将下面脚本保存为：

`submit-test.ps1`

使用前需要修改：

- `$baseUrl`
- `$assignmentId`
- `$studentCount`

第一次建议：

- `$studentCount = 5`

确认无误后再改成 `100`。

```powershell
$baseUrl = "http://localhost:8080"
$assignmentId = 3
$studentCount = 5
$studentPrefix = "ptest_stu"

$jobs = 1..$studentCount | ForEach-Object {
    $i = $_

    Start-Job -ArgumentList $i, $baseUrl, $assignmentId, $studentPrefix -ScriptBlock {
        param($i, $baseUrl, $assignmentId, $studentPrefix)

        $username = "{0}{1}" -f $studentPrefix, ("{0:D3}" -f $i)
        $password = "123456"

        try {
            $loginBody = @{
                username = $username
                password = $password
            } | ConvertTo-Json

            $loginResp = Invoke-RestMethod `
                -Uri "$baseUrl/api/login" `
                -Method Post `
                -ContentType "application/json" `
                -Body $loginBody

            $token = $loginResp.data.token

            $cluster = $i % 5

            $mainCode = @"
public class Main {
    public static void main(String[] args) {
        int x = $i;
        if (x > $cluster) {
            System.out.println(x + $cluster);
        } else {
            System.out.println($cluster - x);
        }
    }
}
"@

            $helperCode = @"
public class Helper {
    public static int add(int a, int b) {
        return a + b + $cluster;
    }
}
"@

            $submitBody = @{
                entries = @(
                    @{
                        filename = "Main.java"
                        content  = $mainCode
                    },
                    @{
                        filename = "Helper.java"
                        content  = $helperCode
                    }
                )
            } | ConvertTo-Json -Depth 5

            $submitResp = Invoke-RestMethod `
                -Uri "$baseUrl/api/student/assignments/$assignmentId/submissions/text" `
                -Method Post `
                -Headers @{ satoken = $token } `
                -ContentType "application/json" `
                -Body $submitBody

            [PSCustomObject]@{
                username     = $username
                ok           = $true
                submissionId = $submitResp.data.id
                version      = $submitResp.data.version
                parseOkFiles = $submitResp.data.parseOkFiles
            }
        }
        catch {
            [PSCustomObject]@{
                username = $username
                ok       = $false
                error    = $_.Exception.Message
            }
        }
    }
}

$results = $jobs | Wait-Job | Receive-Job
$jobs | Remove-Job

Write-Host "===== 提交结果统计 ====="
$results | Group-Object ok | Select-Object Name, Count

Write-Host "===== 失败记录 ====="
$results | Where-Object { -not $_.ok } | Format-Table -AutoSize

Write-Host "===== 成功记录（前10条） ====="
$results | Where-Object { $_.ok } | Select-Object -First 10 | Format-Table -AutoSize
```

## 13. 第八步：教师发起查重 Job

将下面脚本保存为：

`run-job-test.ps1`

使用前需要修改：

- `$baseUrl`
- `$assignmentId`
- `$teacherUsername`
- `$teacherPassword`

```powershell
$baseUrl = "http://localhost:8080"
$assignmentId = 3
$teacherUsername = "你的教师账号"
$teacherPassword = "你的教师密码"

$loginBody = @{
    username = $teacherUsername
    password = $teacherPassword
} | ConvertTo-Json

$loginResp = Invoke-RestMethod `
    -Uri "$baseUrl/api/login" `
    -Method Post `
    -ContentType "application/json" `
    -Body $loginBody

$teacherToken = $loginResp.data.token

$jobBody = @{
    thresholdScore = 80
    topKPerStudent = 10
} | ConvertTo-Json

$jobResp = Invoke-RestMethod `
    -Uri "$baseUrl/api/teacher/assignments/$assignmentId/plagiarism-jobs" `
    -Method Post `
    -Headers @{ satoken = $teacherToken } `
    -ContentType "application/json" `
    -Body $jobBody

$jobId = $jobResp.data.id

Write-Host "创建成功，Job ID = $jobId"

do {
    Start-Sleep -Seconds 2

    $statusResp = Invoke-RestMethod `
        -Uri "$baseUrl/api/teacher/plagiarism-jobs/$jobId" `
        -Method Get `
        -Headers @{ satoken = $teacherToken }

    $status = $statusResp.data.status
    $done = $statusResp.data.progressDone
    $total = $statusResp.data.progressTotal

    Write-Host ("当前状态: {0} 进度: {1}/{2}" -f $status, $done, $total)

} while ($status -eq "QUEUED" -or $status -eq "RUNNING")

Write-Host "最终状态: $status"

if ($status -eq "DONE") {
    $reportResp = Invoke-RestMethod `
        -Uri "$baseUrl/api/teacher/plagiarism-jobs/$jobId/report" `
        -Method Get `
        -Headers @{ satoken = $teacherToken }

    Write-Host ("相似对数量: {0}" -f $reportResp.data.pairs.Count)
}
else {
    Write-Host "查重失败，请检查数据库或后端日志"
}
```

## 16. 如何判断这次测试是否通过

## 16.1 提交阶段通过标准

满足以下条件即可认为提交阶段通过：

- 100 个学生大部分成功提交
- `submission` 条数正确
- `submission_file` 条数正确
- `submission_profile` 条数正确

## 16.2 查重阶段通过标准

满足以下条件即可认为查重阶段通过：

- Job 创建成功
- Job 状态从 `QUEUED -> RUNNING -> DONE`
- `progressTotal = 4950`
- `progressDone` 最终等于 `4950`
- `similarity_pair` 与 `similarity_evidence` 正常生成

## 19. 常见失败原因与排查方法

## 19.1 学生登录失败

可能原因：

- 学生账号未插入成功
- 密码错误
- 账号被禁用

排查 SQL：

```sql
SELECT username, password, role, status
FROM users
WHERE username LIKE 'ptest_stu%';
```

---

## 19.2 学生提交失败

可能原因：

- 学生未加入班级
- assignmentId 错误
- token 未传
- 提交内容格式错误

排查 SQL：

```sql
SELECT * FROM sys_class_student WHERE student_id = 某学生ID;
SELECT * FROM assignment_class WHERE assignment_id = 你的assignmentId;
```

---

## 19.3 查重 Job 卡住或失败

可能原因：

- 后端异常
- 数据库写入失败
- 文件存储路径错误

检查：

- 后端控制台日志
- `plagiarism_job.error_msg`

---

## 19.4 progressTotal 不是 4950

可能原因：

- 并不是 100 个有效最新提交
- 有提交 `parseOkFiles = 0`
- 有学生不在同一班级
- 有提交不是 `is_latest = 1`
