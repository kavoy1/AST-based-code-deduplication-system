# 教师端首页重设计方案

- 日期：2026-04-13
- 主题：教师端首页 UI 重设计（教学运营驾驶舱）
- 状态：已完成设计确认，待用户审阅

## 1. 背景

当前教师端首页已经具备基础数据展示能力，但页面整体仍偏向“普通后台统计页”：

- 顶部欢迎区占据较大面积，但缺少真正的控制台气质
- 指标卡、图表卡和右侧模块在视觉上较为平均，缺少主次节奏
- 主图表目前更像单一数据展示，而不是“教学运行态势总览”
- 查重模块虽然存在，但没有形成足够强的风险监控感

本次重设计不改动教师端首页的业务定位，而是将其升级为一个更具产品感、更具中控视角的“教学运营驾驶舱”。

## 2. 设计目标

### 核心目标

- 强化首页的“教学运行态势”表达，而不是泛化统计页
- 提升页面高级感和产品感，摆脱默认后台模板观感
- 让教师在首屏快速理解当前教学运行状态
- 保留现有核心信息入口，但重构视觉优先级

### 明确选择

基于用户确认，本次设计方向固定为：

- 风格方向：数据中控型
- 视觉底色：浅色中控
- 页面定位：教学运营驾驶舱
- 首页主焦点：教学整体运行态势
- 主图表：作业进度总览
- 第二重点模块：查重风险分布

## 3. 非目标

本次设计不包含以下内容：

- 不重构教师端完整信息架构
- 不新增复杂业务流程
- 不扩展到班级管理、作业管理等全部页面的完整改版
- 不改变首页已有核心导航入口含义
- 不设计仅适合宣传图的高装饰化页面

## 4. 总体设计概念

本页的目标不是“更漂亮的后台首页”，而是让教师在进入系统后看到一个具备运行感、监控感和决策感的工作台。

整体概念定义为：

`浅色教学运营驾驶舱`

关键词：

- 专业
- 克制
- 高级
- 有控制感
- 有图表中心
- 风险信息明确

页面视觉应更接近成熟 SaaS 的教学控制台，而非普通管理系统。

## 5. 页面结构

整体采用三层布局：

1. 左侧固定导航
2. 顶部总控栏
3. 主内容区

主内容区遵循 `8:4` 的主次关系：

- 左侧作为主图表区，承载教学整体运行态势
- 右侧作为风险监控与运行提示区

## 6. 模块设计

### 6.1 左侧导航

目标是从“后台菜单”升级为“控制塔侧栏”。

设计要求：

- 使用较窄的深石墨灰侧栏
- 图标统一为细线风格
- 当前激活态使用高亮竖条或轻微内发光，不使用笨重圆底
- 导航结构保持轻量，突出首页、班级、作业、通知、设置

期望感受：

- 更专业
- 更稳
- 不喧宾夺主

### 6.2 顶部总控栏

顶部区域从大面积欢迎条改为三段式控制头部：

- 左侧：页面主标题 `教学运行驾驶舱`
- 中部：日期、周次或当日轻状态信息
- 右侧：快捷动作入口，如 `班级管理`、`作业管理`

补充策略：

- 保留问候语，但降为辅助信息
- 顶部需要更像控制面板，不像空白横幅
- 顶部信息密度略高于当前版本，但视觉仍需保持克制

### 6.3 第一行指标区

第一行保留四个卡位，但视觉权重不再完全平均。

建议结构：

- 班级总数
- 学生总数
- 进行中作业
- 运行关注项（强化卡）

其中前三张为基础指标卡，第四张为强化卡。

强化卡内容不是单纯数字，而是合成态势，例如：

- 未提交提醒
- 即将截止
- 待判定风险

设计要求：

- 指标卡采用更利落的面板语言
- 数字要更有力量感
- 标签文案要克制
- 支持轻趋势标记，但不做过度动效

### 6.4 主图表：作业进度总览

这是整页视觉中心。

设计定位：

- 展示不同作业的提交进度与状态
- 帮助教师快速识别哪些作业推进良好，哪些作业存在提交压力

不再采用单一班级维度的竖向柱图，而改为更适合多任务对比的总览表达。

推荐图表形态：

- 横向堆叠条形图

每个作业显示：

- 作业名称
- 已提交人数
- 未提交人数
- 提交率
- 状态标签（进行中 / 即将截止 / 已截止）

这样处理的优势：

- 更符合“教学态势总览”
- 更适合对多个作业做横向比较
- 信息表达比当前版本更有运营感

### 6.5 右上模块：查重风险分布

该模块的目标不是单纯展示环图，而是形成“风险监控”印象。

建议结构：

- 环图：展示风险分布
- 风险分类：待确认 / 高风险 / 已排除
- 总任务数
- 待处理数

设计要求：

- 高风险信息需要被清晰看见，但只在局部使用暖色
- 环图只是视觉入口，不能成为唯一信息载体
- 模块应具备警报盘或监控面板的气质

### 6.6 右下模块：运行提示 / 风险情报

右下区不建议继续使用弱信息密度图表，而是改成情报摘要区。

可承载内容：

- 高风险相似对数量
- 待人工确认数量
- 本周已完成查重任务数
- 最近一次查重完成时间

设计目标：

- 与右上风险分布形成配套关系
- 提供图表之外的可执行信息
- 提升右侧区域的实用价值与完成度

## 7. 视觉语言

### 7.1 色彩方向

采用浅色中控体系，不走柔和卡片风。

基础配色方向：

- 背景：浅灰蓝 / 雾面金属感
- 主品牌色：雾蓝、冷蓝
- 强调色：深海军蓝
- 风险高亮：局部暖橙或暖红
- 中性信息：冷灰

原则：

- 页面整体保持浅色、清洁、高级
- 风险色只在重点位置使用
- 不使用过于甜、软、发粉的蓝紫体系

### 7.2 面板语言

卡片与图表容器统一为轻量面板系统：

- 弱阴影
- 细描边
- 中等圆角
- 低对比高光

需要避免：

- 厚重白块
- 过度圆润
- 默认组件库质感

### 7.3 排版层级

排版应体现明显的控制台层级：

- 页面标题：稳重、紧凑、权重高
- 指标数字：大而有力量
- 二级标题：清晰但不过分抢眼
- 辅助文案：更轻、更短

整体原则：

- 标题控制感强
- 辅助文案低存在感
- 数字信息具备压舱石作用

### 7.4 图表气质

图表需摆脱默认图表库观感。

具体要求：

- 已提交使用主蓝
- 未提交使用浅灰蓝
- 风险图使用更清晰的等级配色
- Tooltip 采用精致小面板样式
- 网格线弱化，但保持读取性

## 8. 交互原则

- 指标卡 hover 时仅做轻抬升与边框亮起
- 主图表 hover 时展示更精致的详情 Tooltip
- 风险区的高风险数字可以增强，但不能让整页变成报警界面
- 顶部总控栏可轻微吸顶，增强操作连续性

交互整体基调应偏专业、平稳，而非花哨。

## 9. 响应式规则

本页面从设计阶段即考虑桌面端自适应，不做只适合静态截图的布局。

### 主设计稿

- 桌面主稿宽度：1440

### 压缩验证稿

- 次级验证稿宽度：1280

### 布局规则

- 左侧导航固定
- 顶部总控栏始终保留
- 主内容区保持主次分明
- 在较窄桌面下，右侧模块允许下移，不强行保持满高并排

### 模块规则

- 指标卡优先保留一行四卡结构
- 若宽度不足，可通过强化卡视觉重心拉开层级
- 主图表区必须始终是首屏视觉中心

## 10. Figma 落地建议

### 10.1 基础变量

建议优先建立以下变量命名：

- `bg/base`
- `panel/base`
- `panel/strong`
- `text/primary`
- `text/secondary`
- `brand/blue`
- `risk/high`
- `risk/pending`
- `risk/cleared`

### 10.2 核心组件

建议先抽离以下 Figma 组件：

- 侧栏导航项
- 顶部快捷按钮
- 指标卡
- 强化态势卡
- 图表面板
- 风险统计卡
- 状态标签

### 10.3 设计产出建议

本轮建议至少产出：

- 1440 宽首页主稿
- 1280 宽压缩适配稿
- 侧栏导航组件
- 指标卡组件
- 风险面板组件

这样后续如果继续延展班级管理页、作业管理页，可直接复用样式系统与组件。

## 11. 验收标准

如果设计完成后满足以下条件，则本次首页重设计目标成立：

- 用户第一眼感知到这不是普通后台，而是教学运营驾驶舱
- 页面首屏能清晰表达“作业进度总览”是主信息
- 右侧风险模块具备足够明确的风险监控感
- 顶部、指标区、图表区和风险区形成清晰主次
- 页面在 1280 宽下仍保持稳定的信息结构
- 整体气质更高级、更专业、更产品化

## 12. 后续建议

在本页完成后，适合继续统一升级的页面包括：

- 班级管理页
- 作业管理总览页
- 查重结果页

这样可以把首页建立的驾驶舱语言延展成完整教师端视觉系统，而不是只优化单页。

## 13. 备注

本方案基于现有教师端业务边界整理，不改变当前功能闭环，只重塑首页的视觉表达、信息主次与组件体系。
